Namma Yatri — Product Specifications & Feature Documentation
=============================================================

BOOKING FLOW
============

Rider Booking Flow (Current):
1. Open app → Location auto-detected via GPS
2. Enter destination in search bar
3. View fare estimate (transparent, no hidden charges)
4. Confirm booking → Nearest available driver notified
5. Driver accepts → Rider sees driver name, photo, vehicle, plate number
6. Real-time GPS tracking on map
7. Trip starts → Timer and distance tracked
8. Trip ends → Fare calculated → Payment via UPI or cash
9. Both rider and driver rate each other (1–5 stars)

Driver Acceptance Flow:
1. Driver receives ride request notification (sound + vibration)
2. 30-second window to accept or decline
3. If declined or timeout → Next nearest driver notified
4. Driver navigates to pickup using in-app maps (Google Maps integration)
5. Driver marks "Arrived at pickup"
6. Rider boards → Driver starts trip
7. End trip at destination → Earnings credited instantly

FARE CALCULATION
================
Formula: Base Fare + (Distance Rate × km) + (Time Rate × minutes)

Auto-rickshaw (Bengaluru 2025):
- Base fare: ₹30
- Per km: ₹15
- Per minute (traffic): ₹1.5
- Minimum fare: ₹50
- No surge pricing (unlike Ola/Uber)

Note: Fare structure follows Karnataka government's meter fare regulations.
Namma Yatri does NOT add commission or service fees on top of meter fare.

SAFETY FEATURES
===============
Current Safety Implementation:
1. SOS Button
   - Single tap sends emergency alert
   - Shares live location with 3 emergency contacts
   - Calls nearest police station (planned)

2. Trip Sharing
   - Share live trip link with family/friends
   - Link shows: driver name, vehicle, current location, destination, ETA

3. Driver Verification
   - Aadhaar-based identity verification
   - Vehicle RC (Registration Certificate) verification
   - Police clearance certificate
   - Annual re-verification required

4. In-App Safety Guidelines
   - Pre-trip checklist for riders
   - Safety tips displayed before night trips (after 10 PM)

Planned Safety Features (Q1 2026):
- Audio recording of trips (stored encrypted, accessed only if complaint)
- Women-preferred driver option
- Virtual number masking for calls between rider and driver

RATING SYSTEM
=============
Rider Rating:
- Rated by driver after each trip (1–5 stars)
- Factors: Punctuality at pickup, behavior, accurate destination
- Riders below 3.5 rating see warning

Driver Rating:
- Rated by rider after each trip (1–5 stars)
- Factors: Punctuality, cleanliness, driving quality, politeness
- Drivers below 4.0 rating get coaching alert
- Drivers below 3.5 rating suspended pending review

API ARCHITECTURE (ONDC Layer)
==============================
Namma Yatri implements the Beckn Protocol for ONDC:

Search (Discovery):
POST /search
{
  "context": { "domain": "mobility", "action": "search" },
  "message": {
    "intent": {
      "fulfillment": {
        "start": { "location": { "gps": "12.9716,77.5946" } },
        "end": { "location": { "gps": "12.9352,77.6245" } }
      }
    }
  }
}

Select (Choose provider):
POST /select → Driver list with fare estimates

Init (Confirm booking):
POST /init → Booking initiation

Confirm:
POST /confirm → Trip confirmed

Track:
POST /track → Live GPS updates

Cancel:
POST /cancel → Cancellation

Rating:
POST /rating → Post-trip rating
